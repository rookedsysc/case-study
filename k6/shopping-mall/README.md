# shopping-mall k6 테스트

## 비관적 락 테스트 결과

`coupon-only-load.js` 기준으로 쿠폰 발행 API에 대해 비관적 락 환경에서 부하 테스트를 수행했다.

### 결과 요약

- 쿠폰 발행 응답 허용 체크 성공률은 `3%` (`779 / 21,980`) 수준이었다.
- 전체 HTTP 요청 실패율은 `85.48%` (`23,042 / 26,955`)였다.
- `http_req_duration` 의 `p(95)` 는 `1m 0s` 로 측정되었다.
- `iteration_duration` 의 `p(95)` 는 `59.86s` 로 측정되었다.
- 처리량은 `73.89 req/s`, iteration 처리량은 `7.54 it/s` 수준이었다.

### 해석

- 비관적 락이 걸린 상태에서 동시성이 매우 높아지면 응답 지연이 크게 증가했다.
- 허용 응답 비율이 매우 낮고 HTTP 실패율이 높아 현재 설정에서는 안정적으로 처리하지 못했다.
- 특히 `p(95)` 응답 시간이 1분 수준까지 올라가므로, 락 경합과 서버 자원 병목을 함께 점검할 필요가 있다.

### 후속 확인 포인트

- 애플리케이션 로그에서 실제 실패 상태코드와 예외 유형을 확인한다.
- DB 락 대기 시간, 커넥션 풀, Tomcat 스레드 사용량을 함께 점검한다.
- VU 수와 이벤트 수량을 단계적으로 조정해 병목 구간을 다시 측정한다.
